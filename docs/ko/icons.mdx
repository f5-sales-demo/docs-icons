문서 테마는 여러 팩에서 수천 개의 아이콘에 접근할 수 있도록 제공합니다. 모든 팩은 동일한 패턴을 사용합니다: 팩의 `Icon.astro` 컴포넌트를 임포트하고 이름으로 아이콘을 참조합니다.

## 렌더링 방식

모든 아이콘 팩은 인라인 SVG를 렌더링하는 패키지별 `Icon.astro` 컴포넌트를 사용합니다. 각 팩은 고유한 임포트를 가집니다:

```astro
---
// Starlight 내장
import { Icon } from '@astrojs/starlight/components';

// 기타 팩 — 각각 고유한 임포트를 가짐
import LucideIcon from '@f5xc-salesdemos/icons-lucide/Icon.astro';
import MdiIcon from '@f5xc-salesdemos/icons-mdi/Icon.astro';
import F5xIcon from '@f5xc-salesdemos/icons-f5xc/Icon.astro';
---

<Icon name="star" />
<LucideIcon name="rocket" />
<MdiIcon name="database" />
<F5xIcon name="bot-defense" />
```

아이콘은 `currentColor`를 사용하며 특별한 처리 없이 라이트 모드와 다크 모드에 자동으로 적응합니다.

## 다크/라이트 모드 동작

모든 `Icon` 컴포넌트는 `currentColor`를 사용하여 인라인 SVG를 렌더링합니다. 현재 테마에 자동으로 적응하므로 특별한 CSS나 클래스 처리가 필요하지 않습니다.

F5 XC 서비스 아이콘은 라이트 모드와 다크 모드에 자동으로 적응하는 CSS 사용자 정의 속성을 사용하며, 특별한 클래스 처리가 필요하지 않습니다. 다른 다중 색상 아이콘(예: HashiCorp Flight 컬러 변형)의 경우, `.icon-card-image`의 `no-invert` 클래스가 다크 모드에서 원래 색상을 보존합니다.

| IBM 디자인 언어 | Carbon | 2,582개의 엔터프라이즈급 아이콘 |
