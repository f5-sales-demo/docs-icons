Tres paquetes de iconos proporcionan iconos de servicio a color para documentar infraestructura en la nube. A diferencia de los paquetes de iconos monocromáticos, estos preservan los colores originales del proveedor para diagramas de arquitectura.

## Paquetes disponibles

| Paquete | Iconos | Tamaño predeterminado | Página |
|---------|--------|----------------------|--------|
| AWS Architecture | 885 | 48x48 | [AWS](/aws/) |
| Azure Architecture | 606 | 18x18 | [Azure](/azure/) |
| GCP Architecture | 216 | 24x24 | [GCP](/gcp/) |

## Uso

```astro
---
import AwsIcon from '@f5xc-salesdemos/icons-aws/Icon.astro';
import AzureIcon from '@f5xc-salesdemos/icons-azure/Icon.astro';
import GcpIcon from '@f5xc-salesdemos/icons-gcp/Icon.astro';
---

<AwsIcon name="lambda" />
<AzureIcon name="virtual-machine" />
<GcpIcon name="cloud-storage" />
```

## Renderizado de color

Estos paquetes de iconos utilizan los colores originales del proveedor (no `currentColor`). La clase `no-invert` en las tarjetas de iconos preserva los colores en modo oscuro. Cuando utilice estos iconos fuera del diseño de cuadrícula, no se necesita ningún tratamiento especial — los colores están integrados en el marcado SVG.

Consulte la página dedicada de cada paquete para ver cuadrículas de iconos organizadas por categoría de servicio.

Explorar conjuntos completos de iconos: [Iconos de AWS](https://aws.amazon.com/architecture/icons/) | [Iconos de Azure](https://learn.microsoft.com/en-us/azure/architecture/icons/) | [Iconos de GCP](https://cloud.google.com/icons)
